\section*{Introducción del capítulo}
El presente capítulo aborda las particularidades del sistema de gestión a desarrollar. Para registrar las principales características se hace uso de los \ac{rf} y \ac{rnf}. Estos describen las funcionalidades y atributos de calidad que debe poseer el software. En el capítulo \ref{c:chapter1}, se seleccionó la metodología \ac{xp} como guía para el desarrollo del software; por lo tanto, se utilizan las \ac{hu} como herramienta para una descripción detallada de los \ac{rf} y la confección del plan de iteraciones. Mediante el uso de este último, se procederá a la estimación del tiempo requerido para la culminación del desarrollo del sistema y, con el uso de patrones de diseño, se facilitará la posterior descripción de las tarjetas \ac{crc}.

\section*{Conclusiones del capítulo}
Al finalizar el presente capítulo se arriba a las siguientes conclusiones parciales:

\begin{itemize}
	\item El modelado del proceso de negocio permitió definir las actividades a automatizar dentro del mismo.
	\item La especificación de los requisitos mediante historias de usuario de conjunto con un listado hizo posible tener una visión clara de los objetivos de la implementación y permitió confeccionar un plan de iteraciones basado en la prioridad de los requisitos definida por el cliente de conjunto con el equipo de desarrollo.
	\item El empleo de patrones y estándares permitió llevar a cabo una implementación estable con buena comunicación dentro del equipo de desarrollo, lo cual tributó a aumentar la velocidad de desarrollo del equipo calculada para cada iteración.
	\item Se obtuvieron las tarjetas CRC correspondientes a cada una de las clases.
	\item Finalmente, se definieron las tareas de ingeniería y los estándares de codificación que servirán de guía para alcanzar un producto no solo funcional, sino de código legible y escalable.
\end{itemize}

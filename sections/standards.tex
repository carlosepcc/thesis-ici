\section{Estándares de codificación}
Entendemos como estándar de código a un conjunto de convenciones establecidas de ante mano (denominaciones, formatos, etc.) para la escritura de código \citep{vera2019mejores}.

\paragraph{Comentario de código:}
\begin{itemize}
    \item Los comentarios deben ser oraciones completas.
    \item Si un comentario es una frase u oración su primera palabra debe comenzar con mayúscula a menos que sea un identificador que comience con minúscula.
\end{itemize}

\paragraph{Máxima longitud de las líneas:}

\begin{itemize}
    \item Se limitarán todas las líneas a un máximo de 150 caracteres.
    \item Dentro de paréntesis, corchetes o llaves se puede utilizar la continuación implícita para cortar las líneas largas.
\end{itemize}

\paragraph{Líneas en blanco:}
\begin{itemize}
    \item Las definiciones de métodos dentro de una clase deben separarse por una línea en blanco.
    \item Se puede utilizar líneas en blanco escasamente para separar secciones lógicas.
\end{itemize}

\paragraph{Importaciones:}
\begin{itemize}
    \item Las importaciones deben estar en líneas separadas.
    \item Las importaciones siempre deben colocarse al comienzo del archivo.
\end{itemize}

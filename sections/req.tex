\subsection{Requisitos no funcionales}
Los \ac{rnf}, como su nombre sugiere, son aquellos requerimientos que no se refieren directamente a las funciones específicas que proporciona el sistema, sino a las propiedades emergentes de este como la fiabilidad, el tiempo de respuesta y la capacidad de almacenamiento. De forma alternativa definen las restricciones del sistema \citep{sommerville2011software}.\\
Los requisitos no funcionales de la aplicación a desarrollar son:
\\
\noindent Usabilidad:
\begin{RNF}
\item El sistema debe ser una aplicación web
\item La interfaz del sistema debe ser intuitiva y fácil de usar
\end{RNF}
Hardware y software:
\begin{RNF}[resume]
\item El sistema operativo del servidor debe tener instalado la máquina virtual de Java 
\item El ordenador donde se ejecute el servidor debe poseer como mínimo 4GB de \ac{ram} y 10GB de almacenamiento.
\item El cliente del sistema se podrá ejecutar en los principales navegadores web: Google Chrome y navegadores basados en \gls{chromium}, Mozilla Firefox y Safari.
\end{RNF}
Seguridad:
\begin{RNF}[resume]
\item Solo se podrá acceder al sistema después de estar autenticado con usuario y contraseña vólidos
\item Solo se podrán usar las funcionalidades del sistema en dependencia de si el usuario autenticado posee los permisos requeridos para la misma
\end{RNF}
Diseño e implementación:
\begin{RNF}[resume]
\item Usar JavaScript como lenguaje de programación del lado del cliente.
%\item Usar VueJS como marco de trabajo de desarrollo del lado del cliente.
\item Usar Java como lenguaje de programación del lado del servidor.
\item Usar Spring Boot como \gls{framework} de desarrollo del lado del servidor.

\item Usar la metodología de desarrollo de software \ac{xp}
\end{RNF}

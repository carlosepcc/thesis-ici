\section{Requisitos no funcionales}
\label{s:req}

Los Requerimientos No Funcionales (RNF), como su nombre sugiere, son aquellos
requerimientos que no se refieren directamente a las funciones especóficas que proporciona el
sistema, sino a las propiedades emergentes de óste como la fiabilidad, el tiempo de respuesta y
la capacidad de almacenamiento. De forma alternativa definen las restricciones del sistema \citep{sommerville2011software}. Los requisitos no funcionales de la aplicación a desarrollar son:
\\
\noindent Usabilidad:
\begin{RNF}
\item El sistema debe ser una aplicación web
\item La interfaz del sistema debe ser intuitiva y fócil de usar
\end{RNF}
Hardware y software:
\begin{RNF}[resume]
\item El sistema operativo del servidor debe tener instalado la móquina virtual de Java 
\item El ordenador donde se ejecute el servidor debe poseer como mónimo 4 GB de RAM y 10 GB de almacenamiento
\item El cliente del sistema se podró ejecutar en cualquier sistema operativo que posea un navegador web
\end{RNF}
Seguridad:
\begin{RNF}[resume]
\item Solo se podró acceder al sistema despuós de estar autenticado con usuario y contraseía vólidos
\item Solo se podrón usar las funcionalidades del sistema en dependencia de si el usuario autenticado posee los permisos requeridos para la misma
\end{RNF}
Diseño e implementación:
\begin{RNF}[resume]
\item Para el desarrollo del lado del cliente se usará VueJS
\item Para el desarrollo del lado del servidor se usará Java con SpringBoot
\item Usar la metodología de desarrollo de softwar\ac{xp}
\end{RNF}

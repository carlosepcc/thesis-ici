 \section{Introducción}
 
 El presente capítulo abordan las particularidades del sistema de gestión a desarrollar. Para registrar las principales características se hace uso de los \ac{rf} y \ac{rnf}. Estos describen las funcionalidades y atributos de calidad que debe poseer el software. 
 	
 En el capítulo \ref{c:chapter1}, se seleccionó la metodología XP como guía para el desarrollo del software; por lo tanto, se utilizan las \ac{hu} como herramienta para una descripción detallada de los \ac{rf} y la confección del plan de iteraciones. Mediante el uso de este último, se procederá a la estimación del tiempo requerido para la culminación del desarrollo del sistema y, con el uso de patrones de diseño, se facilitará la posterior descripción de las tarjetas \ac{crc}.

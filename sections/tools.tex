\section{Herramientas y tecnologías a utilizar}

Para el desarrollo de cualquier aplicación, es necesario utilizar diferentes técnicas como: los patrones de diseño o las metodologías de desarrollo de software, además del uso de distintas herramientas como los compiladores o editores de códigos. Aunque a simple vista parezca que la selección de las tecnologías para desarrollar aplicaciones es fácil, es totalmente lo contrario; para su correcta selección, es necesario ver el problema a resolver desde diferentes ángulos y posibles situaciones futuras. El presente epígrafe aborda alguna de las diferentes herramientas que dan solución a la problemática planteada.


%
\subsection{Herramientas y software}
\begin{description}
    \item[Microsoft Visual Studio Code (v1.72.2)] Es un editor de código simplificado y gratuito con soporte para operaciones de desarrollo como: depuración, ejecución de tareas y el control de versiones. Provee las herramientas necesarias para que un desarrollador pueda realizar un ciclo rápido de codificación, ejecución y depuración, dejando que los procesos de trabajos más complejos sean desarrollados en un IDE \citep{vscode}. En la solución se utiliza como editor de código e \ac{ide}.

    \item[\ac{vp} 17.0 CE] Es una herramienta de \ac{case} que brinda ayuda para el desarrollo de programas informáticos, desde la planificación, pasando por el análisis y el diseño, hasta la generación del código fuente de los programas y la documentación. \citep{vp}. En la solución se utiliza como herramienta de apoyo en la elaboración de los artefactos ingenieriles como el diagrama entidad-relación.

    \item[Git] Git es un sistema de control de versiones de código abierto y de libre uso, diseñado para manejar desde grandes hasta pequeños proyectos, con gran eficiencia y velocidad \citep{git}.\\ Un sistema de control de versiones es aquel que registra los cambios realizados en uno o varios archivos a lo largo del tiempo, de modo que se puedan recuperar versiones específicas más adelante, así como auditar y comparar cambios \citep{gitCVS}. En la solución se utiliza para mantener el control, realizar reestructuración al código sin poner en peligro la estabilidad alcanzada gracias al uso de rama, así como para lograr una buena comunicación y agilidad en el desarrollo de la aplicación cuando los miembros del equipo no comparten el mismo espacio físico y trabajan sobre el mismo código.

    \item[Mozilla Firefox (107.0)] Es un navegador web de código abierto y software libre, desarrollado por la Fundación Mozilla \citep{firefox}.
    \item[Google Chrome (104.0)] Es un navegador web desarrollado por Google y compilado con base en componentes de
        código abierto como el motor de representación de WebKit y su estructura de desarrollo de aplicaciones (Framework) \citep{chrome}. Tanto Firefox como este último navegador se usan, durante el desarrollo de la solución, para ejecutar la aplicación cliente, \emph{debugging} y realización de pruebas.
\end{description}

\subsection{Lenguajes de programación, marcos de trabajo y librerías}
\begin{description}

    \item[\ac{js} ES6]
        Es el lenguaje de programación más utilizado hoy. Junto a HTML y CSS, le da vida a la gran mayoría de los sitios Web que visitamos. Pero \ac{js}, ya no es exclusivo de la Web: proyectos como NodeJS, Electron y React Native lo llevaron al ámbito de los servidores, los programas de escritorio y las aplicaciones móviles. Además, cada día surgen nuevos frameworks (como los líderes y archienemigos React y Angular) para expandir las posibilidades de JavaScript \citep{luna2019javascript}. Se utiliza como lenguaje de programación del lado del cliente en el desarrollo de la solución.

    \item[VueJS v3]
        Es un marco progresivo basado en el lenguaje de programación JavaScript. Las funcionalidades principales, que son el renderizado y el sistema de componentes, se encuentran en una biblioteca con las funcionalidades justas para que funcione, eso permite un renderizado en un tiempo corto. Las demás funcionalidades se consiguen añadiendo bibliotecas externas \citep{vuejs2019estudio}. Es usado como framework de desarrollo del lado del cliente debido a la experiencia previa del equipo de desarrollo en el trabajo con este framework, su simplicidad, enfoque a componentes y posibilidad de desarrollar tanto aplicaciones de una sola página como aplicaciones web progresivas.

    \item[Quasar Framework v2]
        Es un marco de código abierto basado en Vue.js para crear aplicaciones, con una sola base de código, e implementarlo en la web como SPA, PWA o SSR; en una aplicación móvil, usando Cordova para iOS y Android; y en una aplicación de escritorio, utilizando Electron para Mac, Windows y Linux \citep{vuejs2019estudio}. Se utiliza en el desarrollo de la solución por sus amplias posibilidades con el maquetado rápido de la aplicación, lo que permite mostrar un primer entregable en forma de prototipo al cliente muy rápido e implementar mejoras de usabilidad con pocas líneas de código, entre otras cosas, porque utiliza el estándar \emph{material-design} para el diseño de los componentes de \ac{gui} de su biblioteca, lo cual garantiza componentes con alto grado de usabilidad sin necesidad de configuración. Además, se usa por ser el único marco gratuito con soporte completo para la versión 3 de Vue.js.

    \item[NodeJS 16.0]
        Es un entorno de ejecución para JavaScript construido con el motor de JavaScript V8 de Chrome. Node.js usa un modelo de operaciones E/S sin bloqueo y orientado a eventos, que lo hace liviano y eficiente. El ecosistema de paquetes de Node.js, npm, es el ecosistema más grande de librerías de código abierto en el mundo \citep{sanchez2021sistemas}. Se usa como entorno de ejecución para el desarrollo de la aplicación cliente por su amplio soporte.

    \item[\ac{npm} v16.0]
        Es el administrador de paquetes por defecto para el lenguaje de programación JavaScript. Es el ecosistema más grande de librerías de código abierto en el mundo. El registro de paquetes de npm contiene más de un millón de paquetes de código abierto. Estos paquetes se usan principalmente para la gestión de dependencias en aplicaciones de JavaScript en el navegador, en aplicaciones de Node.js y en aplicaciones de escritorio basadas en Node.js \citep{node}. Se usa como administrador de paquetes de NodeJS para el desarrollo de la aplicación cliente por su amplia biblioteca.

    \item[Java v11]
        Es un lenguaje de programación orientado a objetos y una plataforma de desarrollo desarrollada por Sun Microsystems a principios de los años 90. Aún Java es el primer lenguaje orientado a objetos enseñado en escuelas debido a su rigor y riqueza funcional. La sintaxis de Java es similar a la de los lenguajes C y C++ pero evita características semánticas que los hacen complejos, confusos y no securizados. \citep{groussard2012java}. Es usado en el desarrollo de la solución como lenguaje de programación del lado del servidor.

    \item[Spring Boot v2.2.5.RELEASE]
        Es un framework desarrollado para el trabajo con Java como lenguaje de programación. Se trata de un entorno de desarrollo de código abierto y gratuito. Facilita la creación de aplicaciones autónomas basadas en Spring. Toma una postura sobre esta plataforma y las librerías de terceros para que se pueda empezar con el mínimo de complicaciones. \citep{haro2019spring}. Se usa como marco de trabajo del lado del servidor por sus facilidades en cuanto a la poca necesidad de configuración para comenzar un nuevo proyecto.
    \item[PostgreSQL v15.0]
        También llamado Postgres, es un sistema de gestión de bases de datos relacional orientado a objetos y de código abierto. Mediante un sistema denominado MVCC (Acceso concurrente multiversión, por sus siglas en inglés) PostgreSQL permite que mientras un proceso escribe en una tabla, otros accedan a la misma tabla sin necesidad de bloqueos. Los usuarios pueden crear sus propios tipos de datos, los que pueden ser por completo indexables \citep{soberon2020psqlmongo}. Se usa para gestionar la base de datos del sistema propuesto.

    \item[pgAdmin 4 v6.11]
        Es una herramienta de gestión para PostgreSQL y bases de datos relacionales derivadas. Es una reimplementación, con tecnologías web, de pgAdmin. Se puede ejecutar como una aplicación web o de escritorio. Se usa para la gestión de la base de datos del la solución \citep{soberon2020psqlmongo}. Se usa como herramienta para la gestión de la base de datos PostgreSQL del sistema propuesto.
\end{description}

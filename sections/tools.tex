\section{Herramientas y tecnologías a utilizar}

Para el desarrollo de cualquier aplicación, es necesario utilizar diferentes técnicas como: los patrones de diseño o las metodologías de desarrollo de software, además del uso de distintas herramientas como los compiladores o editores de códigos. Aunque a simple vista parezca que la selección de las tecnologías para desarrollar aplicaciones es fácil, es totalmente lo contrario; para su correcta selección, es necesario ver el problema a resolver desde diferentes ángulos y posibles situaciones futuras. El presente epígrafe aborda alguna de las diferentes herramientas que dan solución a la problemática planteada.


%
\subsection{Herramientas y software}
\begin{description}
    \item[Microsoft Visual Studio Code] Es un editor de código simplificado y gratuito con soporte para operaciones de desarrollo como: depuración, ejecución de tareas y el control de versiones. Provee las herramientas necesarias para que un desarrollador pueda realizar un ciclo rápido de codificación, ejecución y depuración, dejando que los procesos de trabajos más complejos sean desarrollados en un IDE \citep{vscode}.
    \item[\ac{vp}] Es una herramienta de \ac{case} que brinda ayuda para el desarrollo de programas informáticos, desde la planificación, pasando por el análisis y el diseño, hasta la generación del código fuente de los programas y la documentación \citep{vp}.
    \item[Git] Git es un sistema de control de versiones de código abierto y de libre uso, diseñado para manejar desde grandes hasta pequeños proyectos, con gran eficiencia y velocidad \citep{git}.\\ Un sistema de control de versiones es aquel que registra los cambios realizados en uno o varios archivos a lo largo del tiempo, de modo que se puedan recuperar versiones específicas más adelante, así como auditar y comparar cambios \citep{gitCVS}.
    \item[Mozilla Firefox] Es un navegador web de código abierto y software libre, desarrollado por la Fundación Mozilla \citep{firefox}.
    \item[Google Chrome] Es un navegador web desarrollado por Google y compilado con base en componentes de
        código abierto como el motor de representación de WebKit y su estructura de desarrollo de aplicaciones (Framework) \citep{chrome}.
\end{description}
%
\subsection{Lenguajes de programación, marcos de trabajo y librerías}
\begin{description}
    \item[\ac{js}] Es el lenguaje de programación más utilizado hoy. Junto a HTML y CSS, le da vida a la gran mayoría de los sitios Web que visitamos. Pero \ac{js}, ya no es exclusivo de la Web: proyectos como NodeJS, Electron y React Native lo llevaron al ámbito de los servidores, los programas de escritorio y las aplicaciones móviles. Además, cada día surgen nuevos frameworks (como los líderes y archienemigos React y Angular) para expandir las posibilidades de JavaScript \citep{luna2019javascript}.
    
    \item[VueJS] Es un framework progresivo basado en el lenguaje de programación JavaScript. Las funcionalidades principales, que son el renderizado y el sistema de componentes, se encuentran en una biblioteca con las funcionalidades justas para que funcione, eso permite un renderizado en un tiempo corto. Las demás funcionalidades se consiguen añadiendo bibliotecas externas \citep{vuejs2019estudio}.\

    \item[Java] Es un lenguaje de programación orientado a objetos y una plataforma de desarrollo desarrollada por Sun Microsystems a principios de los años 90. Aún Java es el primer lenguaje orientado a objetos enseñado en escuelas debido a su rigor y riqueza funcional. La sintaxis de Java es similar a la de los lenguajes C y C++ pero evita características semánticas que los hacen complejos, confusos y no securizados. \citep{groussard2012java}.
    
    \item[NodeJS] Es un entorno de ejecución para JavaScript construido con el motor de JavaScript V8 de Chrome. Node.js usa un modelo de operaciones E/S sin bloqueo y orientado a eventos, que lo hace liviano y eficiente. El ecosistema de paquetes de Node.js, npm, es el ecosistema más grande de librerías de código abierto en el mundo \citep{sanchez2021sistemas}. 
    
    \item[\ac{npm}] Es el administrador de paquetes por defecto para el lenguaje de programación JavaScript. Es el ecosistema más grande de librerías de código abierto en el mundo. El registro de paquetes de npm contiene más de un millón de paquetes de código abierto. Estos paquetes se usan principalmente para la gestión de dependencias en aplicaciones de JavaScript en el navegador, en aplicaciones de Node.js y en aplicaciones de escritorio basadas en Node.js \citep{node}. 
\end{description}
